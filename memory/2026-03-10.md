## 🔄 工作流程改进 - 实时反馈

### 用户反馈
- **问题**: 长时间命令执行时没有任何反馈，用户需要等待完成后才能看到所有输出
- **影响**: 用户体验差，不知道进度如何，像"黑盒"执行
- **最终反馈**: 即使使用了 poll，消息仍然是等任务完成后才发出，用户希望在消息窗口实时看到进度

### 改进方案
✅ **使用 message 工具主动推送消息**：
1. 任务开始时立即发送通知
2. 执行过程中定期发送进度更新
3. 遇到问题立即报告
4. 完成时发送最终结果

### 实现示例
```python
# 立即通知
message(action="send", message="🚀 开始执行任务")

# 执行步骤1
exec("command1")
message(action="send", message="✅ 步骤 1/3 完成")

# 执行步骤2
exec("command2")  
message(action="send", message="✅ 步骤 2/3 完成")

# 最终结果
message(action="send", message="🎉 全部完成！")
```

### 实施标准
| 命令时长 | 处理方式 |
|---------|---------|
| < 10秒 | 直接等待完成 |
| 10-60秒 | 发送 2-3 次更新 |
| > 1分钟 | 每 10-30 秒发送更新 |
