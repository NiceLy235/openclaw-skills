### lerobot-auto-train 添加确认步骤
**时间**: 10:52
**用户需求**: 训练前必须确认参数和配置

**修改内容**:
1. **task_manager.py**:
   - 添加 `--dry-run` 参数
   - 实现 `_show_dry_run()` 方法，展示完整配置预览
   - 包含信息：
     - 🔄 数据集信息 (repo_id, episodes, 位置)
     - 🤖 模型配置 (model_name, policy_type, 预训练权重)
     - 📊 训练参数 (batch_size, steps, frequencies, workers)
     - 💻 执行环境 (脚本位置, conda环境, 设备, 输出目录)
     - 🔧 完整训练命令
   - 格式化输出目录为 `outputs/mylerobot_train/MMDD_HHMM`

2. **SKILL.md 文档更新**:
   - 添加 "⚠️ 训练前确认流程（强制）" 部分
   - 定义 5 步确认流程：
     1. 收集用户需求
     2. 生成配置预览
     3. 展示给用户
     4. 等待用户确认
     5. 提交任务
   - 更新示例会话，展示确认流程

**测试结果**:
- ✅ `--dry-run` 参数工作正常
- ✅ 配置预览格式化输出正确
- ✅ 包含所有必要信息
- ✅ Git commit: f56493a

**使用示例**:
```bash
# 1. 预览配置
python scripts/task_manager.py submit \
  --dataset-repo-id ly/merged \
  --dry-run

# 2. 确认后提交（移除 --dry-run）
python scripts/task_manager.py submit \
  --dataset-repo-id ly/merged
```

**工作流程**:
```
用户请求训练
    ↓
Agent 询问参数
    ↓
Agent 运行 --dry-run
    ↓
展示配置给用户
    ↓
等待用户确认
    ↓
用户确认 → 提交训练
用户拒绝 → 修改参数
```

**安全保障**:
- ✅ Agent 不会在未经确认的情况下提交训练
- ✅ 用户可以看到完整的训练命令
- ✅ 用户可以查看数据集和脚本位置
- ✅ 所有参数透明可见
